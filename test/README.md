# `test` 目录说明

这里用于放自动化测试。

## 当前状态

- 现有 [widget_test.dart](widget_test.dart) 还是 Flutter 模板自带的计数器测试。
- 这个测试和当前记账应用界面不匹配，直接运行大概率不能反映真实功能状态。

## 建议优先补的测试

- 新增支出是否成功写入数据库
- 编辑支出是否正确更新
- 删除支出是否生效
- 按月、按年、自定义区间筛选是否正确
- 分类新增、编辑、删除逻辑是否正常
- 备份与恢复流程的基本可用性

## 推荐方向

- 纯数据逻辑优先做单元测试
- 页面交互做 widget test
- 备份恢复这类流程可考虑做集成测试
